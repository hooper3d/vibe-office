# UI_REVIEW_CHECKLIST.md

Use this checklist before finishing any frontend UI task.

## Layout

- Is everything aligned?
- Is spacing consistent?
- Is there enough breathing room?
- Is the page not overcrowded?
- Are sections clearly separated?
- Are there unnecessary nested cards?

## Visual Hierarchy

- Is the page title clear?
- Is the primary action obvious?
- Are secondary actions visually weaker?
- Is important information easy to scan?
- Is the first screen focused?

## Components

- Are buttons consistent?
- Are cards consistent?
- Are badges consistent?
- Are inputs consistent?
- Are tables readable?
- Did you reuse existing components where possible?

## Color

- Are colors restrained?
- Is the accent color used consistently?
- Are status colors used only for status?
- Are random colors removed?
- Does the page avoid cheap admin-template styling?

## Typography

- Are font sizes consistent?
- Are headings, labels, and body text clearly different?
- Is bold used only where needed?
- Is long text easy to read?

## States

- Is there an empty state where needed?
- Is there a loading state where needed?
- Is there an error state where needed?
- Are disabled states clear?
- Are destructive actions confirmed?

## Responsive

- Does it work on desktop?
- Does it work on tablet?
- Does it work on mobile?
- Is there unwanted horizontal overflow?
- Do tables or dense layouts adapt on small screens?

## Accessibility

- Can users navigate with keyboard?
- Is focus visible?
- Do icon-only buttons have labels?
- Is status not communicated by color alone?
- Are form errors readable and placed near the field?

## Final Rules

If the UI feels noisy, simplify it.

If the page has too many colors, remove colors.

If the page has too many cards, merge sections.

If there are cards inside cards, flatten the structure.

If there is no clear main action, add one.

If the page looks like a generic admin template, improve it.
